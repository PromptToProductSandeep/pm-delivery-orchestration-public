Where rough requirements + attachments land — durable, re-readable, governed.

Strategy & Risks

The critique that keeps the build honest.
Uncomfortable answer first
The load-bearing risk is the copy-paste seam: if a ticket gets created and never recorded, your source of truth lies. The cockpit makes that un-skippable with the awaiting-capture state.

Sequencing

Access first (connect Rovo/Drive), then the un-bypassable governance/enforcement layer, then durable state + audit, then the cockpit UI. The cockpit is a view over the engine — last, not first.

The real engine

The hygiene scanner and the reconciliation engine are the same read-and-compare core pointed at two targets: the rules page (scanner) and the state store (reconciler). Build it once.

Stop rule

Retire the manual run sheet when a second-PM run surfaces no new drift category and passes all five signals. New instances are fine; new categories mean the failure surface is still incomplete.

Net
Build the database + enforcement, run the manual loop to extract the spec, then let the breaks log tell you what to automate. Nothing gets built that the log did not ask for.

MVP PRD

One control plane for delivery orchestration.

Problem & vision

Orchestration is invisible: state lives in chat, approvals are implicit, execution is manual copy-paste. The cockpit makes orchestration visible and gate-driven so the PM manages outcomes while the system manages routing, packets, approvals, and traceability.

MVP scope

  • Dashboard + workstream detail with the state object, timeline, routing, and agent status.
  • Separated approval gates; create gates blocked until readiness; awaiting-capture record step.
  • Recorded output + revision loop per gate; append-only audit, breaks, and run logs.

Governance

Approval of one action never approves another; drafting≠execution; lookup≠application; memory always standalone; a gate cannot advance while a prior action is awaiting capture; Coordinator records before the next gate opens.

Phases

P1 manual cockpit (this). P2 generated packets + partial automation. P3 Coordinator calls Rovo/Drive directly behind hard gates.
